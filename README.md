## Текущие допущения

* Комиссия при создании транзакции начисляется для обоих типов операций и вычисляется в классе `BalanceValidationService`.  Принцип SOLID нарушен осознанно. В рамках текущего ТЗ реализация SRP не является приоритетной.
* В каждой транзакции участвует 3 контрагента:
  * плательщик
  * получатель
  * платформа - получает комиссию за проведение платежей
* При каждой транзакции создается 6 проводок
* Поле `transaction_id` таблицы `balance_transactions` содержит единый для всех шести транзакций идентификатор
* Трансформация статуса транзакции:
  * request
  * pending
  
  В целях упрощения после статуса `pending` возможен статус `complete` или `cancelled`, данные статусы возможны для расширения в `TransactionStatusEnum`
